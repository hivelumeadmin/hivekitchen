Superseding note

The direction, as built out.

The original HTML explored three divergent design directions (D1 Letter, D2 Kitchen Counter, D3 Notebook). Step 9 locked D2 Kitchen Counter as the chassis with selective D1 borrows (Heart Note composer, Lunch Link). This file now shows that decided direction as realized through the component strategy (Step 11), the UX consistency patterns (Step 12 — with the inverted §12.8 cultural recognition), and the responsive plus accessibility commitments (Step 13). Dark-mode-first, WCAG 2.2 AA floor, AAA for .child-scope and safety copy.

Section 3 · Button taxonomy

Five variants, honest copy.

Primary, secondary, tertiary, proposal (Lumi-initiated offers only), destructive (narrowed — red reserved for safety). No "OK / Cancel" — every button names what it does. The proposal variant hovers to --lumi-terracotta-warmed, not honey-amber — honey is reserved for recognition moments.

Section 4 · Thread as spine

The conversation is the spine.

The <ThreadTurn> envelope dispatches on body.kind to five body components (message, plan_diff, proposal, system_event, presence) plus the cultural.proposal body from §12.8.2. Turns ordered by server_seq, never created_at.

Section 5 · Cultural recognition (§12.8 inverted)

Recognition as noticing, not as form field.

No template-picker UI. No weight sliders. Templates are internal priors the agent uses; the only user-visible invocation is conversational ratification. The tier ladder renders differently based on opt-in, provenance, and confidence state — with a ⋯ Why this pick? affordance when Lumi falls back a tier.

L0

Preference memory · no opt-in · works in silence-mode

Maya doesn't eat bell peppers.

L1

Method / ingredient · inferred, no opt-in

Tuesday's rice is pressure-cooked, not boiled.

L2

Meal-pattern · requires opt-in + user used pattern name first (language-discovered gate)

Keeping Jollof Friday.

L3

Family-language / relational · requires active state + memory-node citation

Nani's dal is on the week. Ayaan's pick.

Silence-mode (the default — §12.8.5)

Every household ships in silence-mode out of the box. No cultural copy, no "Celebrating X" strip, no picker. L0 preference memory still works. Warmth comes from relational specificity.

Section 10 · Responsive & accessibility

Galaxy A13. 4G. AA floor.

Mobile-first CSS with logical properties (RTL-safe). Performance anchor is the 2022 Samsung Galaxy A13 on 4G. WCAG 2.2 AA across the product; AAA for .child-scope and safety-critical copy.

Voice accessibility (§13.5)
Every voice turn has a synchronized transcript. No voice-only content, ever. Text-only mode disables TTS without degrading comprehension. Low-confidence STT surfaces an honest error rather than faking a transcript.
Section 11 · Phasing (build order)

Foundation first. Culture second. Never stereotype.

Four Phase 0 gates must land before any component. Silence-mode is the default day-1 experience. Cultural recognition ships in Phase 2 after content pipeline lands.
